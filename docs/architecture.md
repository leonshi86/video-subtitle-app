# 视频字幕工具 - 设计文档

## 系统架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                          主线程 (GUI)                               │

│  │  URL 输入框   │  │  播放器控件   │  │      字幕显示区域        │  │
│  └──────────────┘  └──────────────┘  └──────────────────────────┘  │
│                              ↑ Signal                               │
│                              │                                      │
│  ┌──────────────┐  ┌────────┴─────────┐  ┌──────────────────────┐  │
│  │  进度条显示   │←─│  Signal/Slot    │←─│  导出按钮 (SRT/TXT)  │  │
│  └──────────────┘  └──────────────────┘  └──────────────────────┘  │
└─────────────────────────────────────────────────────────────────────┘
                              ↑ Signal
                              │
┌─────────────────────────────┴───────────────────────────────────────┐
│                        后台工作线程                                  │
│                                                                      │
│  ┌────────────────────┐         ┌────────────────────────────────┐  │
│  │  DownloadWorker    │         │     TranscribeWorker          │  │
│  │  (QThread)         │         │     (QThread)                 │  │
│  │                    │         │                                │  │

```

## 数据流

```
用户输入 URL
    │
    ▼
DownloadWorker.download(url)
    │
    ├── 解析视频信息 (fetch_video_info)
    ├── 调用 yt-dlp 下载
    ├── 实时推送进度 (progress_updated signal)
    │
    ▼
download_finished signal
    │
    ├── 返回 DownloadResult (file_path, title, duration)
    │
    ▼
MainWindow._load_video(file_path)
    │
    ├── 设置 QMediaPlayer 源
    │
    ▼
TranscribeWorker.transcribe(file_path)
    │
    ├── 加载 Faster-Whisper 模型 (CPU/int8)
    ├── 分段转写音频
    ├── 实时推送进度 (progress_updated signal)
    │
    ▼
transcription_done signal
    │
    ├── 返回 List[SubtitleEntry]
    │
    ▼
播放时根据 positionChanged 查找对应字幕
    │
    ▼
用户导出 SRT/TXT 文件
```

## 关键设计决策

### 1. 生产者-消费者模型

**问题**：CPU 转写耗时较长，不能阻塞 UI 线程。

**解决方案**：
- 下载和转写分别在独立的 QThread 中执行
- 通过 PySide6 Signal/Slot 实现线程间通信
- 主线程仅负责 UI 渲染和视频播放

### 2. CPU 优化策略

**问题**：Intel UHD Graphics 无 CUDA 支持，必须纯 CPU 推理。

**解决方案**：
- `device="cpu"` 强制 CPU 模式
- `compute_type="int8"` 使用 8-bit 量化，降低内存占用
- `num_workers=1` 单线程批处理，避免多进程争抢 CPU
- 分段转写（每段 30 秒），降低单次内存峰值

### 3. 字幕同步算法

**问题**：播放器每秒触发多次 positionChanged 信号，需要高效查找对应字幕。

**解决方案**：
- 使用二分查找（O(log n)）定位字幕条目
- 预先生成 `start_sec` 和 `end_sec` 字段
- 校验 `end_sec` 防止跨段漂移

### 4. 错误处理

**下载失败**：
- 捕获子进程异常
- 通过 Signal 推送错误信息到 UI
- 显示用户友好的错误提示

**转写失败**：
- 捕获模型加载异常
- 支持取消操作
- 提供 traceback 便于调试

## 性能优化建议

### 内存优化

1. **模型单例**：同一 size 的模型只加载一次，复用缓存
2. **分段转写**：避免一次性加载整个音频文件
3. **及时释放**：转写完成后调用 `gc.collect()`

### 速度优化

1. **使用小模型**：`tiny` 或 `base` 模型速度更快
2. **仅下载音频**：音频文件更小，转写更快
3. **VAD 过滤**：自动过滤静音片段，减少无效推理

### 用户体验优化

1. **预热模型**：应用启动时后台加载模型
2. **进度显示**：实时显示下载和转写进度
3. **可取消操作**：支持取消下载和转写

## 扩展功能（未来版本）

1. **批量下载**：支持播放列表下载
2. **字幕编辑**：手动修正识别错误
3. **多语言支持**：自动检测语言或手动选择
4. **GPU 加速**：为有独立显卡的用户提供 CUDA 支持
5. **字幕翻译**：集成翻译 API
