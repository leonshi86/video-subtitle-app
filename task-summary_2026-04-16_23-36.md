## 任务：流媒体字幕本地缓存 + 字幕全程同步修复

### 时间
2026-04-16 23:29 ~ 23:36

### 需求 1：流媒体字幕自动缓存

**问题**：流媒体 URL 播放时，视频下载到临时目录（tempfile.mkdtemp），每次播放生成新临时路径，字幕无法复用。

**方案**：
- 新增 subtitle_cache/ 目录（应用根目录下），按视频标题做 key
- 3 个新方法：_subtitle_cache_path(title, ext) / _try_load_cached_subtitle(title) / _save_subtitle_cache(title, entries)
- _on_stream_downloaded：先查缓存 → 命中则 _load_video(skip_subtitle=True) 跳过识别
- _on_transcribe_done：转写完成后写入缓存
- _load_video 新增 skip_subtitle 参数

### 需求 2：字幕全程同步

**问题**：播放时暂停转写线程（_on_state_changed 调 pause()），导致识别进度远落后于播放进度，字幕大面积空白。

**根因分析**：
- 现有同步机制本身是完整的：positionChanged → 二分查找 → 高亮
- 真正的问题是播放时转写被暂停，entries 增长停滞
- LowPriority 线程优先级已足够让出 UI CPU，无需额外暂停

**修复**：去掉 _on_state_changed 中所有 pause()/resume() 调用，转写全程不中断。

### 文件变更
- gui/main_window.py：字幕缓存 3 方法 + _on_stream_downloaded + _load_video(skip_subtitle) + _on_state_changed 去掉暂停转写
